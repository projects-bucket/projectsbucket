Our Workshop Plan

How about we divide the sessions according to a product development plan along with the respective tools for each module. some-good-notes.super.site

  • Starting with validating an idea, looking at competition, research and finalization.
  • Design, high fidelity, information architecture, high fidelity, accessibility and mock ups.
  • Front End
  • Back End
  • Maybe if we have time we can also cover markets, scaling, growth and stuff.

It could be like the days are divided into periods where one could ideate and think and design and develop.

Day 1 - Session 1 : Ideate.

  • Begin with problem identifications.
  • Validate if it's actually a problem.
  • Possible Solutions.
  • Prioritization Frame Works.
  • End with a narrowed down solution.

Day 1 - Session 2 : Design.

  • Feature Identification.
  • Information Architecture.
  • Low Fidelity and Iterations.
  • Design Systems.
  • High fidelity interaction mock ups.
  • Code Generation using tools form mock ups.

Day 2 - Session 3 : Build Front End

  • What's front end.
  • Approach to building.
  • Maybe a possible exercise to build using basics.
  • Frame works and their implementations ( Have to discuss a bit)
  • Build something.
  • Managing code ( Git and Git hub).

Day 2 - Session 4 : Build Back End

  • What's back end.
  • Approach.
  • A little bit of Firebase [ Discuss ].
  • Various Technologies involved
  • Build something.
  • Deploying.
